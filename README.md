Task Manager API - NestJS & PrismaPrueba técnica desarrollada bajo los estándares de Arquitectura Modular de NestJS 11 y Prisma ORM 5.22.0. El sistema implementa un CRUD robusto para la gestión de tareas con persistencia en PostgreSQL y despliegue mediante Docker.
Requisitos PreviosNode.js v20 o superior.Docker & Docker Compose.Postman o herramienta similar para pruebas de API.🛠️ Configuración y Despliegue LocalSiga estos pasos para inicializar el entorno de desarrollo:
1. Variables de EntornoCree un archivo .env en la raíz del proyecto con la siguiente configuración:Fragmento de códigoDATABASE_URL="postgresql://postgres:postgres@localhost:5432/tasks_db?schema=public"
2. Infraestructura (Docker)Levante el contenedor de la base de datos PostgreSQL:Bashdocker-compose up -d
3. Instalación de DependenciasInstale los paquetes necesarios asegurando la compatibilidad de peer dependencies:Bashnpm install --legacy-peer-deps
4. Sincronización de Base de DatosGenere el cliente de Prisma y ejecute las migraciones iniciales:Bashnpx prisma generate
npx prisma migrate dev --name init
5. Ejecución del ServidorInicie la aplicación en modo desarrollo:Bashnpm run start:dev
 Arquitectura y Mejores PrácticasPrincipios SOLID: Separación clara de responsabilidades entre Controladores (rutas) y Servicios (lógica de negocio).Validación DTO (Data Transfer Object): Implementada con class-validator y class-transformer para asegurar la integridad de los datos de entrada.Pipes Globales: Uso de ValidationPipe para el manejo automático de errores de validación.Estabilidad de Entorno: Downgrade estratégico a Prisma v5.22.0 para garantizar compatibilidad total con el motor de ejecución en entornos Windows/Docker.Clean Code: Organización de imports, tipado estricto de TypeScript y eliminación de código muerto.📡 Endpoints de la APIMétodoEndpointDescripciónPOST/tasksCrear una nueva tarea (Valida título y prioridad).GET/tasksListar todas las tareas registradas.GET/tasks/:idObtener el detalle de una tarea específica por ID.PATCH/tasks/:idActualizar campos de una tarea (Título, descripción, estado).DELETE/tasks/:idEliminación lógica/física de una tarea.
