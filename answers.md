Middleware vs Guard vs Interceptor vs Pipe:Middleware: Se ejecuta antes que cualquier cosa. Ideal para logs o manejo de CORS.Guard: Se encarga de la Autorización. Determina si la petición sigue o se bloquea (ej. JWT).Interceptor: Transforma la respuesta o maneja la lógica antes/después de que el método se ejecute (ej. formatear un JSON).Pipe: Se usa para Validación y Transformación de datos de entrada (ej. convertir un ID de string a número).Autorización por Roles: Explica que usarías un Guard personalizado junto con Decoradores de NestJS para marcar qué roles pueden acceder a cada endpoint.Concurrencia (Último producto): Para evitar que dos usuarios compren lo mismo, menciona el uso de Transacciones de base de datos con Pessimistic Locking (Bloqueo pesimista) en PostgreSQL o un bloqueo distribuido usando Redis.
